# 要求落地协议报告

来源：工作区已落地协议整理版。

## 目的
长期要求、配置修改、执行规则，不能只停留在对话里，必须进入可检查、可执行、可复核的落地链路。

## 判断方式
新要求先分类为：
1. 配置变更
2. 执行规则
3. 输出标准
4. 一次性任务

### 必须进入强落地链路的条件
满足以下任一条就必须保存：
- 会影响后续多次执行
- 会改变固定时间、固定窗口、固定格式
- 用户明确要求“记住”“落地”“强配置”“以后都这样”

## 保存方式
要求必须至少保存到以下之一：
- 配置文件
- 检查清单
- 两者同时保存（关键要求推荐）

当前新闻相关已落地文件：
- NEWS_CONFIG.md
- NEWS_EXEC_CHECKLIST.md

## 检查方式
### 落地检查
必须能说明：
1. 这条要求是否已分类
2. 是否已写入文件
3. 写入到哪个文件
4. 是否会在下次执行前被检查到

### 执行前检查
每次正式执行前必须逐项核对清单。

### 执行后复核
正式输出后，回看时间、窗口、格式和是否违反已保存规则。

## 执行方式
1. 接收要求
2. 分类
3. 保存到文件
4. 明确报告保存位置
5. 执行前按清单检查
6. 执行后复核
7. 若发现偏差，优先修改文件/清单，而不是只口头纠正

## 禁止事项
- 只在回复里口头确认，不保存
- 说“记住了”但没有写入文件
- 说“已落地”但说不出保存位置
- 执行前不检查，执行后才补救
