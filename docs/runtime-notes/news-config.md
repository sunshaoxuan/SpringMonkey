# 新闻播报配置（JST）

来源：工作区已落地配置整理版。

## 固定执行时间
- 09:00
- 17:00

## 固定取材时间窗口
### 09:00 播报
- 前一日 17:00 ～ 当日 09:00

### 17:00 播报
- 当日 09:00 ～ 当日 17:00

## 固定分区
- 日本
- 中国
- 国际
- 市场或风险提示

## 强约束
1. 正式播报必须对应 09:00 或 17:00 固定锚点。
2. 必须使用对应固定窗口，不得改成浮动“过去若干小时”。
3. 临时测试窗口不得污染正式播报窗口。
4. 临时额外播报必须明确标记为“测试”或“临时窗口”。

## 排版强约束
1. 只允许一级数字标题：1. 日本 / 2. 中国 / 3. 国际 / 4. 市场或风险提示
2. 标题与时间窗口不得编号
3. 一级标题下必须且只空 1 行
4. 子层级禁止数字编号
5. 主区块之间必须且只空 1 行

## 默认生成流程（两阶段）
1. 候选新闻逐条处理，不再整批一次性交给单个模型直出
2. 单条判断阶段优先使用本地模型
3. 每条候选新闻都要落成结构化记录并追加写入任务临时目录
4. 全部候选处理完成后，先用脚本做去重、归类、时间排序、链接完整性检查
5. 只有在前两阶段完成后，才允许调用 Codex 做最后一次整体格式校准

## 频道输出约束
1. 默认不要把中间过程连续发到频道
2. 除非用户明确要求实时播报，否则只发最终结果
3. 最多保留 1 条“开始执行”消息
4. 最多保留 1 条“完成结果”消息

## 当前状态
- 正常新闻播报暂停
- 在格式化解决方案真正落地并通过检查前，只允许进行实际数据测试（非冒烟）
