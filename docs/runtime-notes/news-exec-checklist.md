# 新闻正式播报执行检查清单（JST）

来源：工作区已落地检查清单整理版。

## 时间锚点检查
- 本次播报锚点必须是 09:00 或 17:00
- 不得误用 08:30 / 16:30 / 17:30 / 浮动当前时间

## 取材窗口检查
### 若为 09:00 播报
- 取材窗口必须是 前一日 17:00 ～ 当日 09:00

### 若为 17:00 播报
- 取材窗口必须是 当日 09:00 ～ 当日 17:00

### 通用要求
- 不得把“过去若干小时”当成正式窗口
- 不得把临时测试窗口混进正式播报

## 内容检查
- 分区固定为：日本 / 中国 / 国际 / 市场或风险提示
- 不能只剩政治经济，也要纳入合适的社会、民生、科技、文化、娱乐、生活类内容
- 不合格原料必须丢弃
- 保留内容必须改写与汇总，不得直接照搬原始标题
- 不得对外暴露后台筛选逻辑

## 两阶段流程检查
- 候选新闻必须逐条处理，不能整批一次性交给单个模型完成
- 单条判断阶段优先使用本地模型，不要先用 Codex
- 每条候选新闻必须落成结构化记录并持续写入本次任务临时文件
- 结构化记录至少包含：是否保留、栏目归属、一句话事实摘要、原文链接、来源名称、时间戳、保留或丢弃原因
- 在最终成稿前，必须先用脚本完成去重、按栏目归类、时间排序、原文链接完整性检查
- Codex 只允许用于最后一轮格式校准，不得重新做整批新闻筛选

## 排版检查
- 只使用一级数字标题：1. 日本 / 2. 中国 / 3. 国际 / 4. 市场或风险提示
- 标题与时间窗口不得编号
- 一级标题下只空 1 行
- 子层级不用数字编号
- 主区块之间只空 1 行
- 不得出现编号嵌套编号

## 发布检查
- 当前输出必须明确区分为：正式播报 / 实际数据测试 / 冒烟测试
- 正式播报前必须完成自检
- 默认不要把中间过程连续发到频道
- 除非用户明确要求实时播报，否则只发最终结果
- 最多只发 1 条开始执行消息和 1 条完成结果消息

## 执行后复核
- 播报时间正确
- 取材时间段正确
- 标号与空行正确
- 若有偏差，优先修配置和清单，而不是只口头纠正
