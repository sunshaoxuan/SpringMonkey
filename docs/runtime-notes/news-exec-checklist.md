# 新闻正式播报执行检查清单（JST）

来源：工作区已落地检查清单整理版。

## 时间锚点检查
- 本次播报锚点必须是 09:00 或 17:00
- 不得误用 08:30 / 16:30 / 17:30 / 浮动当前时间

## 取材窗口检查
### 若为 09:00 播报
- 取材窗口必须是 前一日 17:00 ～ 当日 09:00

### 若为 17:00 播报
- 取材窗口必须是 当日 09:00 ～ 当日 17:00

### 通用要求
- 不得把“过去若干小时”当成正式窗口
- 不得把临时测试窗口混进正式播报

## 内容检查
- 分区固定为：日本 / 中国 / 国际
- 不能只剩政治经济，也要纳入合适的社会、民生、科技、文化、娱乐、生活类内容
- 不合格原料必须丢弃
- 保留内容必须改写与汇总，不得直接照搬原始标题
- 不得对外暴露后台筛选逻辑

## 排版检查
- 只使用一级数字标题：1. 日本 / 2. 中国 / 3. 国际
- 一级标题下只空 1 行
- 子层级不用数字编号
- 主区块之间只空 1 行
- 不得出现编号嵌套编号

## 发布检查
- 当前输出必须明确区分为：正式播报 / 实际数据测试 / 冒烟测试
- 正式播报前必须完成自检

## 执行后复核
- 播报时间正确
- 取材时间段正确
- 标号与空行正确
- 若有偏差，优先修配置和清单，而不是只口头纠正
