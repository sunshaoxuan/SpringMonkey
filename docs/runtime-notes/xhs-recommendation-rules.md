# 小红书推荐文规则

更新时间：2026-05-02（东京时间）

## 工作流

- 每 3 天产出一篇小红书推荐文。
- 每次先写进 Google Docs，供用户讨论、修改、确认。
- 文案和图片都确认没问题后，才进入发布小红书步骤。
- 不自动发布；发布前必须得到用户明确确认。

## 选品与核验

- 推荐具体单品，不写泛泛的清单。
- 涉及 Costco / コストコ 商品时，先核验官方在售信息或可靠来源。
- 商品编号、官网价格、评分等信息可用于内部核验，但不一定写进正文。
- 商品番号通常不要写进正文，除非用户明确要求。

## 图片规则

- 必须使用真图。
- 至少 3 张图：两张以上官网/官方产品图，至少一张网络实拍图。
- 不使用 AI 生成图、示意图、看起来像素材库的假图。
- 不允许有任何可见水印。
- 图片来源要能支撑商品真实性。

## 文案口吻

- 推荐文要像真实的人在生活区分享，不像选题说明、商品卡片、电商详情页或 AI 总结。
- 不要出现“我为什么推荐”“图文结构建议”“结论”“先纠错”“如果你想写”等给用户解释思路的词。
- 不要写成和用户商量、复盘、讨论选题的语气。
- 正文要直接面向读者，像朋友自然分享。
- 可以有“我觉得”“最近看到”“下次逛到可以顺手带”这类自然表达。
- 对读者的称呼可以根据商品受众和场景灵活调整，不要固定一种叫法。
- 面向妈妈/带娃场景时，可以自然使用“宝妈”“溜娃的姐妹们”等；面向男生或男性向商品时，可以用“兄弟们”“老少爷们儿”等；更通用的场景可以用“铁汁们”“各位老铁”“宝贝们”等。
- 这些称呼只是风格参考，实际写作时要根据内容、受众、平台语感自由选择；不要为了用称呼而硬塞。

## 格式规则

- 少用冒号式结构，例如“商品名：xxx”“推荐理由：xxx”。
- 少用项目编号、条列、卡片式信息堆叠。
- 重要信息尽量藏进自然句里。
- 标题不要太像营销标题，也不要过度说明地点和渠道。
- 如果文章主体是中文，正文应该以中文为主，可以灵活混入日文词来形成日本生活语境。

## Costco / コストコ 表达

- 标题不要写“日本 Costco”。用户是在生活区发文，这样会显得像游客视角。
- 正文如果提到 Costco，统一写成「コストコ」。这样日本语境已经足够。
- 不要在正文反复强调“日本 Costco”。
- 购买渠道最多自然提一句即可。

## 话题标签规则

- Google Docs 里话题每个一行，不能全堆在一行。
- 发布到小红书前必须在平台预览里确认话题显示效果；小红书网页端会把已识别的话题组件自动并排展示，普通回车又可能变成段落空行。
- 如果平台无法同时做到“可点击话题 + 每条一行 + 无空行”，优先保持可点击话题与整洁显示，并向用户说明平台限制，不要反复盲改。
- 文章主体是中文，话题也应中文为主。
- 可混少量日文话题，用来体现日本生活和コストコ语境。
- 不要全部写日文，避免显得刻意蹭日区标签。
- 不要再写“日本Costco”这种中英混合硬标签，优先用「コストコ」相关表达。
